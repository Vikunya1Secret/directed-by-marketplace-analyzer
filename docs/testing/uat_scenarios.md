# Сценарии приёмочного тестирования (User Acceptance Testing) – MVP

## Цель
Подтвердить, что веб-сервис «Выгодный Взгляд» соответствует минимально необходимому функционалу и готов к демонстрации куратору.

## Формат сценария
- **ID**: уникальный номер
- **Название**: краткое описание
- **Предусловия**: что должно быть готово перед тестом
- **Шаги**: действия пользователя
- **Ожидаемый результат**: что должно произойти
- **Фактический результат** (заполняется при тестировании)

---

### Модуль 1: Поиск товара

| ID | UAT-01 |
|----|--------|
| **Название** | Успешный поиск по точному названию товара |
| **Предусловия** | Сервис запущен, парсеры Ozon и WB настроены |
| **Шаги** | 1. Ввести в строку поиска «Sony WH-1000XM5»<br>2. Нажать кнопку «Найти» |
| **Ожидаемый результат** | Загружается страница результатов. Отображается не менее 2 предложений (с Ozon и Wildberries). Каждое предложение содержит: цену, срок доставки, рейтинг продавца, кнопку «Перейти». |
| **Фактический результат** | |

| ID | UAT-02 |
|----|--------|
| **Название** | Поиск по артикулу |
| **Предусловия** | Известен артикул товара (например, наушников) |
| **Шаги** | 1. Ввести артикул (цифры/буквы)<br>2. Нажать «Найти» |
| **Ожидаемый результат** | Сервис находит товар (хотя бы на одной площадке). Результат отображается корректно. |
| **Фактический результат** | |

| ID | UAT-03 |
|----|--------|
| **Название** | Поиск товара, которого нет на маркетплейсах |
| **Предусловия** | – |
| **Шаги** | 1. Ввести несуществующий запрос (например, «sdkjhgfdskjhg»)<br>2. Нажать «Найти» |
| **Ожидаемый результат** | Отображается сообщение «Ничего не найдено. Попробуйте изменить запрос.» |
| **Фактический результат** | |

### Модуль 2: Сравнение и результаты

| ID | UAT-04 |
|----|--------|
| **Название** | Отображение всех полей в карточке товара |
| **Предусловия** | Выполнен поиск, есть результаты |
| **Шаги** | 1. Посмотреть на карточку любого товара |
| **Ожидаемый результат** | Присутствуют: название, цена (₽), срок доставки (дни), рейтинг продавца (звёзды или число), «индекс выгоды» (число от 0 до 1), кнопка-ссылка на маркетплейс. |
| **Фактический результат** | |

| ID | UAT-05 |
|----|--------|
| **Название** | Сортировка по индексу выгоды (по умолчанию) |
| **Предусловия** | Есть хотя бы 2 предложения с разными ценами/доставками |
| **Шаги** | 1. Выполнить поиск<br>2. Посмотреть порядок карточек |
| **Ожидаемый результат** | Первый товар имеет наибольший индекс выгоды, последний – наименьший. Индексы отображаются. |
| **Фактический результат** | |

| ID | UAT-06 |
|----|--------|
| **Название** | Переход на маркетплейс по кнопке |
| **Предусловия** | Открыта страница результатов |
| **Шаги** | 1. Нажать кнопку «Перейти» на любом предложении |
| **Ожидаемый результат** | Открывается новая вкладка браузера с URL товара на Ozon или Wildberries (соответственно маркетплейсу). |
| **Фактический результат** | |

### Модуль 3: Настройка весов (персонализация)

| ID | UAT-07 |
|----|--------|
| **Название** | Изменение весов через слайдеры |
| **Предусловия** | Открыта страница результатов после поиска |
| **Шаги** | 1. Найти форму настройки весов (три слайдера: цена, доставка, рейтинг)<br>2. Передвинуть слайдер «цена» на максимум, остальные на минимум<br>3. Нажать «Применить» |
| **Ожидаемый результат** | Сортировка обновляется. Самое дешёвое предложение становится первым, независимо от доставки и рейтинга. |
| **Фактический результат** | |

| ID | UAT-08 |
|----|--------|
| **Название** | Сброс весов к значениям по умолчанию |
| **Предусловия** | Веса изменены |
| **Шаги** | 1. Нажать кнопку «Сбросить» (или «По умолчанию») |
| **Ожидаемый результат** | Слайдеры возвращаются в положение: цена=0.5, доставка=0.3, рейтинг=0.2. Сортировка пересчитывается. |
| **Фактический результат** | |

### Модуль 4: Производительность и стабильность

| ID | UAT-09 |
|----|--------|
| **Название** | Время выполнения поиска |
| **Предусловия** | Очищен кеш браузера |
| **Шаги** | 1. Ввести запрос и нажать «Найти»<br>2. Засечь время от нажатия до полной загрузки результатов |
| **Ожидаемый результат** | Время не превышает 8 секунд (на медленном интернете – до 10). |
| **Фактический результат** | |

| ID | UAT-10 |
|----|--------|
| **Название** | Устойчивость к ошибкам парсинга (один маркетплейс недоступен) |
| **Предусловия** | Симулировать ошибку подключения к Ozon (например, отключить сеть для парсера) |
| **Шаги** | 1. Выполнить поиск |
| **Ожидаемый результат** | Результаты от Wildberries всё равно отображаются. Пользователь видит сообщение: «Данные с Ozon временно недоступны, показаны только предложения с Wildberries». Ошибка не вызывает падения сервиса. |
| **Фактический результат** | |

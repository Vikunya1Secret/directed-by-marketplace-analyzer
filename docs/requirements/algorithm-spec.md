# Алгоритм ранжирования "Индекс выгоды"

## Версия: 1.0 (MVP)

## Цель алгоритма
На основе персональных весов пользователя (важность цены, скорости доставки, рейтинга продавца) рассчитать для каждого товарного предложения единый **индекс выгоды** и отсортировать предложения по убыванию этого индекса (самый выгодный – первым).

## Входные данные
- Список предложений с разных маркетплейсов (Ozon, Wildberries). Каждое предложение содержит:
  - `price` – цена в рублях (число > 0)
  - `delivery_days` – срок доставки в днях (целое число, от 1 до 30)
  - `seller_rating` – рейтинг продавца (число от 0 до 5)
- Веса пользователя:
  - `w_price` – важность цены (от 0 до 1)
  - `w_delivery` – важность скорости доставки (от 0 до 1)
  - `w_rating` – важность рейтинга продавца (от 0 до 1)

## Шаги алгоритма

### 1. Нормализация параметров (приведение к единому диапазону 0..1)
Для каждого параметра используем **минимальное значение среди всех найденных предложений** как референс.

- **Нормированная цена**  
  `norm_price = min_price / price`  
  (где `min_price` – самая низкая цена среди всех предложений в выдаче)

- **Нормированная доставка**  
  `norm_delivery = min_delivery / delivery_days`  
  (где `min_delivery` – минимальный срок доставки среди всех предложений)

- **Нормированный рейтинг**  
  `norm_rating = seller_rating / 5.0`

### 2. Нормализация весов (чтобы их сумма = 1)
Если пользователь не задал веса, используются значения по умолчанию:  
`w_price = 0.5`, `w_delivery = 0.3`, `w_rating = 0.2`.

Если сумма весов не равна 1, они делятся на их сумму:
um_w = w_price + w_delivery + w_rating
w_price_norm = w_price / sum_w
w_delivery_norm = w_delivery / sum_w
w_rating_norm = w_rating / sum_w

### 3. Расчёт индекса выгоды
`benefit = norm_price * w_price_norm + norm_delivery * w_delivery_norm + norm_rating * w_rating_norm`

### 4. Сортировка
Все предложения сортируются по убыванию `benefit`. При равных значениях – более дешёвое предложение идёт первым.

## Пример расчёта (ручной)

### Исходные данные
Предложения:

| Маркетплейс | Цена (руб) | Доставка (дни) | Рейтинг |
|-------------|------------|----------------|---------|
| Ozon        | 25000      | 2              | 4.9     |
| Wildberries | 24000      | 5              | 4.5     |

Пользовательские веса:  
`w_price = 0.4`, `w_delivery = 0.4`, `w_rating = 0.2` (сумма = 1, нормализация не нужна)

### Расчёт
- `min_price = 24000`, `min_delivery = 2`

#### Ozon:
- `norm_price = 24000 / 25000 = 0.96`
- `norm_delivery = 2 / 2 = 1.0`
- `norm_rating = 4.9 / 5 = 0.98`
- `benefit = 0.96*0.4 + 1.0*0.4 + 0.98*0.2 = 0.384 + 0.4 + 0.196 = 0.98`

#### Wildberries:
- `norm_price = 24000 / 24000 = 1.0`
- `norm_delivery = 2 / 5 = 0.4`
- `norm_rating = 4.5 / 5 = 0.9`
- `benefit = 1.0*0.4 + 0.4*0.4 + 0.9*0.2 = 0.4 + 0.16 + 0.18 = 0.74`

### Результат
Рекомендуется **Ozon** (benefit 0.98 > 0.74), несмотря на более высокую цену, из-за быстрой доставки и высокого рейтинга.

## Особенности реализации на MVP
- Нормализация выполняется **каждый раз** при новом поиске (в пределах одной поисковой выдачи).
- Если для какого-то предложения отсутствует рейтинг или доставка, используются значения по умолчанию: рейтинг = 4.0, доставка = 5 дней.
- Все расчёты ведутся в числах с плавающей точкой (float).

## Планируемые улучшения (после MVP)
- Учёт реальной стоимости доставки (не только дней).
- Добавление веса для «надёжности продавца» (процент выкупов, количество отзывов).
- Машинное обучение для корректировки весов на основе обратной связи пользователя.
